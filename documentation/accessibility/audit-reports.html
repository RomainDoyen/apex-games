Rapport initial (baseline) — Apex Games

Date : 2026-02-15

  • Constat : documentation technique créée et adaptée au projet (architecture, features, API). Action : documentation HTML accessible avec navigation, recherche, liens cohérents.
  • Objectif : conserver une base accessible pour le frontend — contrastes suffisants (design sombre), focus visible sur les liens et boutons, libellés explicites (carthes jeux, favoris, backlog).
  • Zones à auditer régulièrement : page d’accueil (tendances), catalogue et filtres, détail jeu, tableau Kanban (drag & drop + alternative clavier), formulaires login/register, états de chargement et messages d’erreur.
