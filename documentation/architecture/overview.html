Architecture — Overview

React 19 Vite NestJS RAWG Supabase Dernière mise à jour : 2026-02-15

Objectif

Une architecture simple et maintenable : frontend React (Vite), API NestJS, données jeux via RAWG, et Supabase pour l’auth et le backlog. Le README principal reste court ; le détail est dans cette documentation.

Structure du repo

apex-games/
├── apex-games-frontend/     (React 19 + Vite — app utilisateur)
├── apex-games-backend/      (NestJS — API REST, proxy RAWG, auth, backlog)
├── documentation/           (ce mini-site statique)
└── README.md

Conventions

  • API REST : endpoints clairs, réponses cohérentes, erreurs structurées.
  • Sécurité : validation, auth (Supabase/JWT), rate limiting (Throttler).
  • Frontend : composants réutilisables, état avec Zustand, appels via Axios.

Stack technique

Frontend

  • React 19, TypeScript
  • Vite (build)
  • React Router, Zustand, Axios
  • Lucide React (icônes)

Backend

  • NestJS (TypeScript)
  • Supabase (auth, données)
  • RAWG (catalogue jeux)
  • Jest (tests)

Configuration (env)

  • Frontend : apex-games-frontend/.env — VITE_RAWG_API_KEY, URL API backend si besoin.
  • Backend : apex-games-backend/.env — Supabase, RAWG, CORS, etc.
