Vue d’ensemble (texte)

React Frontend (apex-games-frontend)
  ├─ Page d’accueil / tendances
  │    └─ Appels API (proxy ou direct) → RAWG
  ├─ Catalogue / découverte
  │    └─ Backend NestJS ou proxy → RAWG (filtres, pagination)
  ├─ Détail d’un jeu
  │    └─ RAWG (id)
  ├─ Favoris
  │    └─ Zustand (local) et/ou Backend → Supabase
  ├─ Backlog Kanban
  │    └─ Backend NestJS → Supabase (backlog)
  └─ Auth (login / register)
       └─ Backend NestJS ↔ Supabase Auth

Flux : Découverte de jeux

  1. Le frontend envoie une requête (liste, recherche, filtres) au backend ou au proxy RAWG.
  2. Le backend peut appeler RAWG avec la clé API, mettre en cache (GameCache) si configuré.
  3. Réponse : liste de jeux (ids, titres, images, métadonnées) affichée dans le catalogue.

Flux : Backlog

  1. L’utilisateur déplace une carte entre colonnes (À jouer, En cours, Terminé).
  2. Le frontend envoie la mise à jour au backend (NestJS → BacklogModule).
  3. Le backend persiste dans Supabase (ou autre store) et renvoie la nouvelle structure.

Flux : Authentification

  1. Inscription / Connexion : formulaire → Backend → Supabase Auth.
  2. Le backend renvoie un token ou des cookies de session.
  3. Les requêtes protégées (backlog, profil) envoient le token dans les headers.
