API — RAWG & Backend NestJS

NestJS RAWG Dernière mise à jour : 2026-02-15

Le frontend peut appeler l’API RAWG (via proxy Vite ou directement avec une clé) et le backend NestJS pour l’auth, le backlog et le cache. Base URL backend : http://localhost:3000 (dev).

Authentification /auth

Rate limiting (Throttler) actif sur ces routes. Réponses : tokens + utilisateur selon la config (JWT / Supabase).

  • POST /auth/register — Inscription (body : email, mot de passe, etc.).
  • POST /auth/login — Connexion.
  • POST /auth/logout — Déconnexion.
  • GET /auth/profile — Profil utilisateur (JWT requis).
  • GET /auth/verify — Vérification du token.
  • POST /auth/forgot-password — Mot de passe oublié (body : email).
  • POST /auth/reset-password — Réinitialisation (body : token, newPassword).

Backlog /backlog

Toutes les routes backlog nécessitent une authentification JWT.

  • POST /backlog — Ajouter un jeu (body : rawgId, status, title, image).
  • GET /backlog — Liste du backlog de l’utilisateur.
  • PATCH /backlog/:rawgId/status — Changer le statut (colonne) d’un jeu.
  • DELETE /backlog/:rawgId — Retirer un jeu du backlog.

RAWG (proxy backend) /rawg

Le backend appelle l’API RAWG avec la clé serveur ; le frontend peut passer par ce contrôleur pour éviter d’exposer la clé.

  • GET /rawg/games — Liste de jeux (dépend de l’implémentation RawgService).
  • GET /rawg/search?query=... — Recherche par texte.
  • GET /rawg/details/:id — Détail d’un jeu par ID RAWG.

Cache & Admin

  • GET /cache-test/:rawgId — Test de cache jeu (GameCacheModule).
  • GET /admin/dashboard, /admin/users, /admin/moderation — Réservés aux rôles admin.

Configuration frontend

Clé RAWG : VITE_RAWG_API_KEY dans apex-games-frontend/.env. Pour utiliser le backend comme proxy, configurer l’URL de l’API dans les services Axios (ex. VITE_API_URL=http://localhost:3000).
