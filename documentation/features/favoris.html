Favoris

Frontend Zustand Dernière mise à jour : 2026-02-15

Objectif

Permettre à l’utilisateur de sauvegarder ses jeux préférés (liste de favoris) et de les retrouver d’une session à l’autre. L’implémentation actuelle est côté client uniquement.

Implémentation

Zustand avec middleware persist : le store (bookmarkStore) contient une liste de favoris (id jeu, nom, image, date d’ajout). Persistance dans localStorage sous la clé bookmark-storage.

API du store

  • addBookmark(game) : ajoute un jeu (évite les doublons).
  • removeBookmark(gameId) : retire un jeu.
  • isBookmarked(gameId) : indique si le jeu est en favoris.
  • clearBookmarks() : vide la liste.

Évolution possible

Pour synchroniser les favoris entre appareils ou les lier au compte utilisateur, un endpoint backend (et Supabase) pourra être ajouté ; le frontend continuera d’utiliser le même store avec une couche API optionnelle.
